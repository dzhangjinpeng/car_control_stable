# 稳定版架构

## 目标

把正式跑车需要的东西拆开，但保持一个仓库可以单独交付：

- C++ 负责实时控制和硬件下发
- Python 负责诊断、校准和远程工具
- 前端只读展示，不直接碰电机

## 目录

```text
car_control_stable/
  core_cpp/      控制核心
  tools_python/  API、遥测、远程输入
  frontend/      只读诊断页面
  configs/       运行配置
  scripts/       构建和现场脚本
  docs/          架构和现场说明
```

## 运行链路

```text
输入源 -> 输入层 -> 安全层 -> 控制器 -> 电机客户端 -> 遥测
```

输入源支持：

- `neutral`
- `demo`
- `gamepad`
- `remote`
- `hybrid`

## 核心循环

每个循环做这几件事：

1. 读取输入
2. 过滤急停和限幅
3. 计算控制命令
4. 下发电机
5. 采样反馈
6. 写 JSONL 遥测

循环本身不做网络请求、不做大文件写入、不做复杂 UI 渲染。

## 校准分层

- `probe`：看实时回读
- `verify`：检查配置和回读
- `drive-direction`：确认驱动电机正反向
- `steer-zero`：写转向零点

## 前端分层

前端只读：

- 总览
- 配置
- 校准报告
- 历史遥测

它通过 Python API 取数据，不直接连电机。
